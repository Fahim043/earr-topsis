Abstract
Fuzzy TOPSIS is widely used for multi-criteria group decision-making because it can rank alternatives under linguistic uncertainty. 
However, the current experimental evidence shows that standard fuzzy TOPSIS and bootstrap-only variants can be highly vulnerable 
when a coordinated group of decision makers intentionally promotes a low-ranked target alternative. This paper draft proposes and 
evaluates a staged reliability-aware ensemble framework, denoted EARR-TOPSIS, that progresses from a conventional fuzzy TOPSIS 
baseline (M1) to bootstrap bagging (M2), reliability filtering (M3), reliability-weighted sampling (M4), cluster-stratified bagging 
(M5), reliability-weighted aggregation (M6), and a multi-signal entropy-, variance-, and clone-aware ensemble method (M7). The 
current benchmark evidence supports the main hypothesis that reliability-aware filtering and weighting substantially reduce targeted 
rank manipulation when malicious decision makers produce structurally distinguishable behavior. Across larger real/pseudo-real 
experiments, M3, M4, M6, and M7 preserve the clean target rank under structured contamination where M1 collapses and M2 remains 
only partially protective. The draft also explicitly defines the threat-model boundary: M7 is not a universal detector of all bias and is 
not guaranteed against adaptive human-mimic attacks that deliberately hide malicious behavior inside human-like variation. The 
method is therefore positioned as a robustness mechanism against distinguishable coordinated manipulation, not as a complete solution 
to strategic adversarial decision-making. Additional prior-art baselines, full attack-fraction curves, repeated-seed statistics, and formal 
significance testing remain necessary before Q1-journal submission.
Keywords: fuzzy TOPSIS; group decision-making; adversarial bias; reliability weighting; ensemble MCDM; entropy; clone 
detection; targeted rank manipulation

1. Introduction
Multi-criteria decision-making (MCDM) methods are frequently used when alternatives must be evaluated across multiple criteria by 
one or more experts. Fuzzy TOPSIS is attractive because triangular fuzzy numbers can encode linguistic uncertainty while retaining a 
transparent distance-to-ideal ranking mechanism. In group decision-making settings, however, the ranking is only as reliable as the 
decision-maker panel. If several decision makers coordinate their ratings, a poor target alternative can be pushed upward even when 
the clean panel would rank it last.

The current experimental outputs motivate a reliability-aware extension to fuzzy TOPSIS. Standard fuzzy TOPSIS (M1) is treated as 
the clean baseline and also as the vulnerable contaminated baseline. A bootstrap ensemble (M2) tests whether variance reduction alone 
is sufficient. Reliability-aware methods (M3-M7) introduce individual, group, sampling, aggregation, and ensemble-level mechanisms 
for reducing the influence of suspicious decision-maker behavior.
The central thesis is deliberately limited: reliability-aware fuzzy TOPSIS can substantially reduce targeted manipulation when 
coordinated decision makers leave detectable statistical traces. The framework is not claimed to defeat all forms of strategic 
adversarial manipulation.
1.1 Contributions
 Formulates targeted rank manipulation in fuzzy group TOPSIS as a robustness problem rather than only as random noise.
 Provides a staged algorithmic family, M1-M7, that separates bootstrap variance reduction from reliability-aware defense.
 Introduces M7 / EARR-TOPSIS as a multi-signal reliability ensemble using entropy, variance consistency, and clone/agreement-
pattern behavior.
 Reports current real/pseudo-real evidence showing that M3, M4, M6, and M7 preserve target rank under structured contamination 
in larger benchmarks, while M1 and M2 remain vulnerable.
 Clarifies that adaptive human-mimic attacks are a deliberate evasion setting, not ordinary human disagreement, and should be 
treated as a limitation rather than a generic failure on human input.
2. Threat model and terminology
The updated report emphasizes that ordinary human variation, subjective human bias, structured coordinated attacks, and adaptive 
human-mimic attacks must not be conflated. This distinction is critical for avoiding overclaiming.

The adaptive human-mimic case should be described as a deliberate evasion strategy. It does not mean that M7 fails whenever 
ordinary humans disagree. Instead, it means that an unsupervised reliability model cannot guarantee detection when malicious decision 
makers are statistically indistinguishable from honest decision makers without external trust, historical behavior, supervised labels, or 
ground truth.

4. Methodology: M1-M7 algorithm family
Figure 3. Algorithmic progression from M1 to M7.
4.1 M1: standard fuzzy TOPSIS
M1 is the classical baseline. It aggregates all decision makers and applies the standard fuzzy TOPSIS ranking pipeline without any 
reliability filtering or adversarial defense.
Strengths. Simple, interpretable, and useful as the clean-reference ranking.
Limitations. Vulnerable to coordinated manipulation because every decision maker contributes directly; extreme ratings can widen 
the fuzzy aggregation envelope.
4.2 M2: bootstrap bagged fuzzy TOPSIS
M2 draws bootstrap samples of decision makers with replacement, runs fuzzy TOPSIS inside each bag, and combines bag rankings or 
closeness coefficients. The updated report states that M2 was corrected from disjoint partitioning to true bootstrap sampling.
Strengths. Reduces random variance and provides an important ensemble baseline.
Limitations. Has no adversarial reliability model; if attackers are sufficiently common, many bags still contain contaminated decision 
makers.

4.3 M3: reliability/outlier-filtered fuzzy TOPSIS
M3 flattens each decision maker rating tensor, measures deviation from a central behavior pattern, and removes suspicious decision 
makers before fuzzy TOPSIS.
Strengths. Strong when biased decision makers are a minority and structurally far from honest decision makers.
Limitations. Can fail under 50% or adaptive human-mimic attacks; depends on the reliability detector.
4.4 M4: reliability-weighted bagging
M4 converts decision-maker reliability into bootstrap sampling probabilities, so more reliable decision makers are sampled more 
frequently.
Strengths. Combines ensemble sampling with reliability and often restores clean target ranks under structured contamination.
Limitations. If reliability scores are wrong, the sampling distribution can reinforce the wrong group.
4.5 M5: cluster-stratified bagging
M5 clusters decision makers by behavior and attempts to reduce the influence of low-reliability clusters. It is a group-level defense 
rather than a purely individual filter.
Strengths. Useful when attackers form a clearly detectable cluster.
Limitations. Current results are inconsistent; it should be treated as an intermediate method, not as the main contribution.
4.6 M6: reliability-weighted aggregation
M6 applies reliability inside each bootstrap bag during TFN aggregation, reducing the contribution of suspicious decision makers even 
when they are sampled.
Strengths. Strong when reliability scores correctly identify suspicious behavior.
Limitations. Still depends on reliability quality and can distort clean cases if reliability is misestimated.
4.7 M7 / EARR-TOPSIS: entropy-aware reliability-weighted ensemble
M7 is the candidate final method. It combines entropy, variance consistency, and clone/agreement-pattern signals into a composite 
reliability model, applies gap-based filtering, uses reliability-weighted sampling and reliability-weighted aggregation, and combines 
bag outputs with reliability-sensitive ensemble weighting.
Strengths. Strong against perfect-clone and structured coordinated attacks; can defend in some majority-attack settings where M1-M6 
fail.
Limitations. Does not defeat all attacks. Adaptive human-mimic attacks remain a deliberate evasion boundary, not ordinary human 
disagreement.

5. Experimental design
Figure 5. Experimental protocol represented from the current benchmark outputs.
The current experiments compare clean and contaminated rankings against the clean M1 reference. The attack is a targeted promotion 
attack: the lowest-ranked target alternative under the clean baseline is manipulated upward by coordinated decision makers.
For crisp datasets that are not native group decision-maker fuzzy TOPSIS datasets, the benchmark creates deterministic pseudo-DM 
fuzzy panels by normalizing values into a 1-9 scale and adding bounded noise. This conversion protocol must be stated clearly in the 
final paper because it affects external validity.
The current benchmark output includes two benchmark settings: (i) max-alternatives = 80, repeats = 5, num-bags = 50, attacker-
fraction = 0.3; and (ii) max-alternatives = 0, repeats = 30, num-bags = 200, attacker-fraction = 0.3. In the 15-DM pseudo-DM settings, 
the effective contamination is 4/15 = 26.7%.

7. Discussion
7.1 Why M1 and M2 fail
M1 treats all decision makers equally, so coordinated attackers directly shift the aggregated fuzzy matrix. M2 uses bootstrap sampling 
but does not know which decision makers are suspicious. If attackers are frequent enough, many bootstrap bags still contain attackers, 
causing partial rather than complete recovery.
7.2 Why reliability helps
M3, M4, M6, and M7 reduce attack influence through filtering, sampling probability, aggregation weights, or a combination of these 
mechanisms. Their success in the larger structured-contamination experiments suggests that malicious behavior leaves detectable 
reliability patterns under the current attack design.
7.3 Why M5 is weaker
M5 is conceptually useful when attackers form a clearly separated cluster, but current results show inconsistent recovery. It should 
remain in the paper as an intermediate method and ablation-like comparison, not as the central contribution.
7.4 Is M7 a failure?
No, not in the larger structured-contamination setting. M7 restores the target rank in the healthcare countries, car evaluation, and 
healthcare-resource 80-alternative benchmarks, and in the full car-evaluation run. However, it does fail or degrade under tiny two-DM 
panels, exact 50% tie-like settings, and adaptive human-mimic attacks. The correct claim is not universal robustness; the correct claim 
is robustness against distinguishable structured coordinated manipulation.
8. Limitations and Q1-readiness gaps
 The current work is not yet Q1-journal ready because prior-art robust MCDM baselines are still missing.
 Repeated-seed statistical reporting must be expanded beyond the current selected runs; confidence intervals or nonparametric 
significance testing should be added.
 Attack-fraction curves are needed for 10%, 20%, 30%, 40%, 50%, and 60% contamination.
 Ablation and sensitivity studies are needed for M7 without entropy, without clone detection, without variance, and with different 
bag counts.
 Real-data conversion from crisp datasets to pseudo-DM fuzzy panels must be carefully justified.
 Criterion-type support remains a known implementation limitation unless all cost criteria are pre-converted.
 Human-mimic attacks, legitimate minority preference cases, exact 50% cases, and false-positive clean-rank distortions should be 
reported as failure modes.
9. Conclusion
This draft presents EARR-TOPSIS as a reliability-aware ensemble extension of fuzzy TOPSIS for adversarially contaminated group 
decision-making. The current evidence supports the hypothesis that standard fuzzy TOPSIS and bootstrap-only TOPSIS are vulnerable 
to targeted rank manipulation, while reliability-aware filtering and weighting can substantially reduce the manipulation when attackers 
exhibit distinguishable structure. M7 contributes a multi-signal reliability model and performs strongly in larger structured-
contamination tests. The manuscript must nevertheless remain careful: M7 is not a universal detector of bias, and adaptive human-
mimic attacks define an important threat-model boundary. The next step is to complete prior-art baselines, attack-fraction curves, 
repeated-seed statistical testing, and ablation studies before journal submission.
Appendix A. Algorithm sketches
Algorithm 1: Standard fuzzy TOPSIS (M1)
Input: alternatives A, criteria C, decision makers D, fuzzy ratings X, fuzzy weights W
1. Aggregate all decision-maker TFNs.
2. Normalize fuzzy decision matrix.
3. Apply fuzzy criterion weights.

4. Compute FPIS and FNIS.
5. Compute D_i^+ and D_i^- for every alternative.
6. Compute CC_i = D_i^- / (D_i^+ + D_i^-).
7. Return alternatives sorted by descending CC_i.
Algorithm 2: M7 / EARR-TOPSIS
Input: fuzzy DM panel X, weights W, number of bags B, reliability parameters
1. Flatten each decision-maker tensor into behavior vector z_k.
2. Compute entropy, variance-consistency, and clone/agreement signals.
3. Combine signals into composite reliability R_k.
4. Apply gap-based filtering to remove low-reliability DMs where appropriate.
5. For each bag b = 1,...,B:
   a. Sample decision makers with probability proportional to R_k.
   b. Aggregate TFNs inside the bag using reliability weights.
   c. Run fuzzy TOPSIS and store CC_i^(b) and bag quality q_b.
6. Compute softmax bag weights alpha_b.
7. Combine bag scores into final ensemble closeness coefficients.
8. Return final ranking.
